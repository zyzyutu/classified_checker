# 涉密信息综合检查系统实验指南（最终版）

## 一、实验目的
开发一个带图形界面（GUI）的集成化工具，实现网页、数据库、本地文件、图片的涉密信息自动检查，并生成规范的 Markdown 检查报告。

## 二、默认路径（程序固定）
- 文档检查目录：F:\保密技术检查\document
- 图片检查目录：F:\保密技术检查\image
- 数据库文件：F:\保密技术检查\baomi.sql

## 三、关键词设置
默认关键词：
涉密、秘密、机密、绝密、保密、泄密

支持：
- 图形界面手动输入、修改关键词
- 多个关键词用逗号分隔

## 四、匹配方式（必须使用）
使用 **正则表达式** 进行模糊匹配，能够识别：
- 带空格的关键词，例如：绝  密
- 带特殊符号的关键词，例如：机@密、秘～密
- 关键词中间插入任意字符的情况

## 五、功能要求

### 1. 网页检查
目标：https://bm.yangyq.net/
- 全站遍历
- 忽略外链
- 统计总数、涉密数、行号

### 2. 数据库检查
- 读取 baomi.sql
- 遍历所有表、所有字段
- 统计记录数、涉密位置

### 3. 文件检查
支持：TXT、DOC、DOCX、XLS、XLSX、PPT、PPTX、PDF
功能：
- 文件头 Magic Number 校验真实类型
- 识别加密文件
- 读取隐藏文件
- 异常不崩溃

### 4. 图片检查
- 本地 OCR 识别
- 不使用任何远程 API
- 统计图片数量、涉密图片路径

### 5. 报告输出
自动生成 Markdown 报告，包含：
- 汇总统计
- 各模块详情
- 涉密精确定位

## 六、界面要求（必须是 GUI 图形界面）
包含：
- 路径显示与修改
- 关键词输入框
- 开始检查按钮
- 进度提示
- 结果展示

## 七、技术要求
- 多线程并发扫描
- WPS 预启动
- pythoncom.CoInitialize()
- 异常捕获，程序不崩溃
