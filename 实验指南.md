# 涉密信息综合检查系统实验指南

## 一、实验目的
开发一个带图形界面（GUI）的集成化工具，实现网页、数据库、本地文件、图片的涉密信息自动检查，并生成规范的 Markdown 检查报告。

## 二、默认路径（程序固定）
- 文档检查目录：`F:\保密技术检查\document`
- 图片检查目录：`F:\保密技术检查\image`
- 数据库连接：GUI 界面输入（地址/用户名/密码/库名）
- 报告输出：`D:\Tools\Claude\classified_checker`

## 三、关键词设置
默认关键词：涉密、秘密、机密、绝密、保密、泄密

支持：
- 图形界面手动输入、修改关键词
- 多个关键词用逗号分隔

## 四、匹配方式（必须使用）
使用 **正则表达式** 进行模糊匹配，能够识别：
- 带空格的关键词，例如：绝　密
- 带特殊符号的关键词，例如：机@密、秘～密
- 关键词中间插入任意字符的情况（每个字符间允许插入 0~2 个任意字符）

## 五、功能要求

### 1. 网页检查
目标：https://bm.yangyq.net/

**已实现功能：**
- BFS 全站遍历，支持配置爬取深度（0~10，默认 5）
- 忽略外链，只检查同域名页面
- **多线程并行爬取**（可配置 1~16 线程，默认 6），显著提升速度
- **增量检测**：启用后利用 HTTP 缓存机制（ETag / Last-Modified / 内容 MD5 哈希）判断页面是否更新，跳过未变化的页面
- 缓存文件 `web_cache.json` 自动保存，支持跨次检查对比
- **缓存涉密标记**：匹配到涉密内容的页面在缓存中标记 `has_match`，增量模式下跳过的涉密页面仍会在报告中展示
- **跳过页面保留链接**：增量模式跳过的页面从缓存中提取链接，保证 BFS 深度遍历不中断
- 统计总页面数、涉密页面数、跳过数、新增/更新数、缓存已有涉密页数
- 涉密精确定位：URL、行号、匹配关键词、内容摘要

### 2. 数据库检查
**已实现功能：**
- **GUI 界面输入连接信息**：数据库地址、用户名、密码（密码框隐藏显示）
- 支持检查全部数据库或指定单个数据库（留空=全部，输入库名=指定）
- **多库并行检查**（ThreadPoolExecutor），提升检查速度
- **INFORMATION_SCHEMA 批量获取元数据**：2 条 SQL 替代 N 条 DESCRIBE，获取表行数、列信息、主键列
- **REGEXP 粗筛 + Python 正则精确匹配**：SQL 端保证字符顺序，Python 端模糊匹配确认
- **自动检测表主键**：通过 `INFORMATION_SCHEMA.KEY_COLUMN_USAGE` 获取主键列，复合主键自动拼接，无主键时回退行号
- **跳过数值字段**（INT/FLOAT/DECIMAL 等），只检查文本类型字段
- **同行去重合并**：同一记录同一字段的多个关键词合并为一条，按 `(表名, 字段, 记录ID)` 去重
- **SQL 注入防护**：库名校验 `^[a-zA-Z0-9_]+$`，REGEXP 特殊字符转义
- 逐表统计：每个表的记录总数、涉密记录数、字段数
- 统计总记录数、涉密表数、粗筛候选数
- 涉密精确定位：表名、字段名、记录 ID、匹配关键词

### 3. 文件检查
支持格式：TXT、DOC、DOCX、XLS、XLSX、PPT、PPTX、PDF、ZIP、RAR、7Z

**已实现功能：**
- **Magic Number 文件头校验**：识别文件真实类型，检测扩展名与实际格式不一致的情况
- **加密文件识别**：
  - DOCX/XLSX/PPTX：三层检测（ZIP flag_bits + EncryptedPackage 流 + 核心内容流缺失）
  - DOC/XLS/PPT：OLE2 EncryptionInfo 流检测
  - PDF：PyPDF2 `is_encrypted` 属性 + 异常捕获
  - 压缩包：ZIP flag_bits 预检、RAR/7z `needs_password()` 预检 + 解压异常关键词匹配
- **损坏文件诊断**：提取失败时自动诊断是加密、损坏还是缺少解析库；PDF 损坏单独检测
- **隐藏文件检测**：通过 Windows API（`GetFileAttributesW`）识别隐藏文件，Linux 下检测 `.` 前缀
- **压缩包支持**：自动解压 ZIP/RAR/7Z，递归检查内部文件（最大嵌套深度 3 层）
- **压缩包炸弹防护**：限制嵌套深度 + `visited_archives` 集合防循环引用
- **多线程并行检查**（可配置线程数）
- **COM 应用缓存**：Word/WPS COM 实例全局缓存，避免每文件重启（2~5 秒/文件 → 首次启动后几乎为零）
- **COM 线程安全**：`threading.RLock()` 可重入锁 + 每线程 `pythoncom.CoInitialize()`
- 多种文本提取方案：COM 接口 → 库解析 → 二进制暴力提取（兜底）
- 支持 TXT 多编码自动识别（UTF-8/GBK/GB2312/Latin-1）
- 统计文件总数、各类型文件数、涉密文件数、加密文件数、损坏文件数、隐藏文件数、压缩包数
- 涉密精确定位：文件路径、行号、文件类型、匹配关键词、内容摘要

### 4. 图片检查
**已实现功能：**
- 本地 OCR 识别（不使用远程 API）
- **OCR 引擎**：RapidOCR（ONNX Runtime），自动检测 DirectML（AMD/Intel）> CUDA（NVIDIA）> CPU 加速
- **多线程并行 OCR**（可配置线程数），充分利用 GPU 并行能力
- **文字位置检测**：根据 OCR 文字框纵坐标判断位置（上部/中部/下部）
- **MD5 缓存**：以图片文件路径+大小+修改时间为 key 缓存 OCR 结果，相同图片跳过识别
- **置信度过滤**：OCR 识别结果置信度低于 0.7 的自动丢弃，减少误报
- 缓存文件 `img_cache.pkl`（pickle 格式）自动保存，支持跨次检查复用
- 递归扫描目录下所有图片（PNG/JPG/JPEG/BMP/TIFF/GIF）
- 统计图片总数、涉密图片数、缓存命中数
- 涉密精确定位：所在目录、文件名、匹配关键词、OCR 识别文本、位置（上/中/下）

## 六、报告输出
自动生成 Markdown 格式报告，包含：
- 汇总统计表（各模块检查总数、涉密数量、备注）
- 网页检查详情（增量统计：跳过数、新增/更新数；缓存涉密页面列表）
- 数据库检查详情（逐表统计 + 涉密匹配详情）
- 文件检查详情（类型统计、加密/损坏/隐藏文件列表、压缩包数、涉密匹配详情）
- 图片检查详情（类型统计、OCR 引擎信息、置信度阈值、涉密图片列表）

**报告格式优化：**
- **路径固定宽度**：文件路径截断为 40 列视觉宽度（CJK 字符占 2 列），超长路径显示为 `前缀/.../文件名`
- **关键词截断**：超过 5 个关键词时显示前 5 个 + `等N个`
- **内容摘要**：短文本（≤120字）直接展示，长文本截取关键词周围 ±40 字上下文

## 七、界面要求（GUI 图形界面）
**已实现功能：**
- 深色主题界面，大字体显示
- 各模块输入框可手动修改（目录/网址/数据库连接信息/关键词/爬取深度/线程数）
- **数据库连接**：地址 / 用户名 / 密码 三个输入框，密码隐藏显示
- **检查范围**：留空检查全部数据库，输入库名检查指定库
- 增量检查开关（复选框）
- 输入校验（URL 格式、数据库名格式、目录是否存在、深度范围）
- 开始检查 / 停止 / 生成报告 / 清空日志 按钮
- **立即停止**：点击停止后中断当前模块，不等待完成
- **断点续查**：停止后点击开始，跳过已完成的模块继续检查
- 进度条实时显示
- 日志区域实时输出（彩色高亮：INFO/WARN/ERROR/SUCCESS/MATCH）

## 八、技术要求
- 多线程并发扫描（网页爬取 + 文件检查 + 图片 OCR 均支持可配置线程数）
- WPS/Word COM 接口调用通过 `pythoncom.CoInitialize()` 初始化
- COM 操作使用 `threading.RLock()` 可重入锁保证线程安全，COM 应用实例全局缓存
- 异常捕获，程序不崩溃（每个模块独立容错）
- 守护线程 + 轮询停止信号，实现立即停止
- SQL 注入防护：库名正则校验 + REGEXP 特殊字符转义
- 数据库连接信息 GUI 输入，不硬编码
- OCR 置信度过滤（阈值 0.7），减少低质量识别的误报
- 图片 OCR 缓存（pickle 格式），避免重复识别
- 网页增量缓存涉密标记，跳过页面保留链接保证 BFS 完整性

## 九、安全设计
| 安全措施 | 说明 |
|---------|------|
| SQL 注入防护 | 库名正则校验 `^[a-zA-Z0-9_]+$` + REGEXP 转义 |
| 密码不硬编码 | GUI 输入，不存储在配置文件中 |
| Magic Number 校验 | 防止文件扩展名欺骗 |
| 隐藏文件检测 | Windows 隐藏属性 + Linux 点前缀 |
| 压缩包炸弹防护 | 限制嵌套深度 3 层 + 循环引用检测 |
| 加密文件识别 | ZIP flag_bits / EncryptedPackage / 核心流 / OLE2 / PDF 属性 / RAR/7z 预检 |
| 输入校验 | URL 格式、数据库名格式、目录存在性 |

## 十、性能优化
| 优化手段 | 位置 | 效果 |
|---------|------|------|
| `requests.Session()` 连接池 | 网页检查 | 减少 TCP 握手开销 |
| 多线程并行爬取 | 网页检查 | 6 线程并行，速度提升约 6 倍 |
| HTTP 条件请求（增量检测） | 网页检查 | 未变化页面不下载 |
| REGEXP 粗筛 + 正则精筛 | 数据库检查 | SQL 端过滤，减少数据传输 |
| INFORMATION_SCHEMA 批量查询 | 数据库检查 | 2 条 SQL 替代 N 条 DESCRIBE |
| 跳过数值字段 | 数据库检查 | 减少无用查询 |
| 多库并行检查 | 数据库检查 | ThreadPoolExecutor 并行 |
| COM 应用实例缓存 | 文件检查 | 避免每文件重启 Word/WPS（2~5s → ~0s） |
| RLock 可重入锁 | 文件检查 | 避免死锁，允许嵌套调用 |
| Magic Number 校验 | 文件检查 | 快速判断文件类型 |
| 压缩包并行解压 | 文件检查 | 多个压缩包同时处理 |
| GPU 加速 OCR | 图片检查 | DirectML/CUDA 加速图片识别 |
| 图片多线程 OCR | 图片检查 | 多张图片同时识别 |
| OCR 缓存（pickle） | 图片检查 | 相同图片跳过 OCR，直接复用结果 |
| OCR 置信度过滤 | 图片检查 | 丢弃低质量识别结果，减少误报 |
