# 涉密信息综合检查系统实验指南

## 一、实验目的
开发一个带图形界面（GUI）的集成化工具，实现网页、数据库、本地文件、图片的涉密信息自动检查，并生成规范的 Markdown 检查报告。

## 二、默认路径（程序固定）
- 文档检查目录：`F:\保密技术检查\document`
- 图片检查目录：`F:\保密技术检查\image`
- 数据库连接：localhost / root / baomi
- 报告输出：`D:\Tools\Claude\classified_checker`

## 三、关键词设置
默认关键词：涉密、秘密、机密、绝密、保密、泄密

支持：
- 图形界面手动输入、修改关键词
- 多个关键词用逗号分隔

## 四、匹配方式（必须使用）
使用 **正则表达式** 进行模糊匹配，能够识别：
- 带空格的关键词，例如：绝　密
- 带特殊符号的关键词，例如：机@密、秘～密
- 关键词中间插入任意字符的情况（每个字符间允许插入 0~5 个任意字符）

## 五、功能要求

### 1. 网页检查
目标：https://bm.yangyq.net/

**已实现功能：**
- BFS 全站遍历，支持配置爬取深度（0~10，默认 5）
- 忽略外链，只检查同域名页面
- **多线程并行爬取**（可配置 1~16 线程，默认 6），显著提升速度
- **增量检测**：启用后利用 HTTP 缓存机制（ETag / Last-Modified / 内容 MD5 哈希）判断页面是否更新，跳过未变化的页面
- 缓存文件 `web_cache.json` 自动保存，支持跨次检查对比
- **缓存涉密标记**：匹配到涉密内容的页面在缓存中标记 `has_match`，增量模式下跳过的涉密页面仍会在报告中展示
- **跳过页面保留链接**：增量模式跳过的页面从缓存中提取链接，保证 BFS 深度遍历不中断
- 统计总页面数、涉密页面数、跳过数、新增/更新数、缓存已有涉密页数
- 涉密精确定位：URL、行号、匹配关键词、内容摘要

### 2. 数据库检查
**已实现功能：**
- 直连 MySQL 数据库（host/user/password/数据库名均可配置）
- 密码支持环境变量读取（`CLASSIFIED_CHECKER_DB_PASS`），不硬编码
- 遍历所有表、所有字段、所有记录
- **两阶段查询优化**：SQL LIKE 粗筛 → Python 正则精确匹配，大幅减少查询量
- **跳过数值字段**（INT/FLOAT/DECIMAL 等），只检查文本类型字段
- **SQL 注入防护**：`_escape_like()` 转义 LIKE 特殊字符（`%`、`_`、`\`）
- **逐表统计**：每个表的记录总数、涉密记录数、字段数
- 统计总记录数、涉密表数、粗筛候选数
- 涉密精确定位：表名、字段名、记录 ID、匹配关键词

### 3. 文件检查
支持格式：TXT、DOC、DOCX、XLS、XLSX、PPT、PPTX、PDF、ZIP、RAR、7Z

**已实现功能：**
- **Magic Number 文件头校验**：识别文件真实类型，检测扩展名与实际格式不一致的情况
- **加密文件识别**：通过 ZIP 加密标志位（`flag_bits & 0x1`）、OLE2 加密信息流（`EncryptionInfo`）、PyPDF2 检测加密文件并跳过
- **损坏文件诊断**：提取失败时自动诊断是加密、损坏还是缺少解析库
- **隐藏文件检测**：通过 Windows API（`GetFileAttributesW`）识别隐藏文件，Linux 下检测 `.` 前缀
- **压缩包支持**：自动解压 ZIP/RAR/7Z，递归检查内部文件（最大嵌套深度 3 层）
- **压缩包炸弹防护**：限制嵌套深度 + `visited_archives` 集合防循环引用
- **加密压缩包检测**：ZIP/RAR/7Z 均支持密码检测，加密压缩包标记跳过
- **多线程并行检查**（可配置线程数），COM 操作（DOC/PPT）通过 `threading.Lock` 保证安全
- **COM 线程安全**：每个线程独立 `pythoncom.CoInitialize()`，`threading.local()` 存储线程状态
- 多种文本提取方案：COM 接口 → 库解析 → 二进制暴力提取（兜底）
- 支持 TXT 多编码自动识别（UTF-8/GBK/GB2312/Latin-1）
- 统计文件总数、各类型文件数、涉密文件数、加密文件数、损坏文件数、隐藏文件数、压缩包数
- 涉密精确定位：文件路径、行号、文件类型、匹配关键词、内容摘要

### 4. 图片检查
**已实现功能：**
- 本地 OCR 识别（不使用远程 API）
- 引擎优先级：RapidOCR（ONNX Runtime） → pytesseract（Tesseract）
- **GPU 加速**：自动检测 DirectML（AMD/Intel）> CUDA（NVIDIA）> CPU
- **多线程并行 OCR**（可配置线程数），充分利用 GPU 并行能力
- **MD5 缓存**：以图片文件内容 MD5 为 key 缓存 OCR 结果，相同图片跳过识别（改名/移动不影响）
- **置信度过滤**：OCR 识别结果置信度低于 0.7 的自动丢弃，减少误报
- 缓存文件 `img_cache.json` 自动保存，支持跨次检查复用
- 递归扫描目录下所有图片（PNG/JPG/JPEG/BMP/TIFF/GIF）
- 统计图片总数、涉密图片数、缓存命中数
- 涉密精确定位：所在目录、文件名、匹配关键词、OCR 识别文本

## 六、报告输出
自动生成 Markdown 格式报告，包含：
- 汇总统计表（各模块检查总数、涉密数量、备注）
- 网页检查详情（增量统计：跳过数、新增/更新数；缓存涉密页面列表）
- 数据库检查详情（逐表统计 + 涉密匹配详情）
- 文件检查详情（类型统计、加密/损坏/隐藏文件列表、压缩包数、涉密匹配详情）
- 图片检查详情（类型统计、OCR 引擎信息、置信度阈值、涉密图片列表）

## 七、界面要求（GUI 图形界面）
**已实现功能：**
- 深色主题界面，大字体显示
- 各模块输入框可手动修改（目录/网址/数据库名/关键词/爬取深度/线程数）
- 增量检查开关（复选框）
- 输入校验（URL 格式、数据库名格式、目录是否存在、深度范围）
- 开始检查 / 停止 / 生成报告 / 查看历史 / 清空日志 按钮
- **立即停止**：点击停止后中断当前模块，不等待完成
- **断点续查**：停止后点击开始，跳过已完成的模块继续检查
- 进度条实时显示
- 日志区域实时输出（彩色高亮：INFO/WARN/ERROR/SUCCESS/MATCH）

## 八、数据持久化
- 检查结果自动保存到 `check_results.json`（最新覆盖模式，同一文件只保留最新结果）
- 点击"查看历史"可查看上次检查的汇总结果

## 九、技术要求
- 多线程并发扫描（网页爬取 + 文件检查 + 图片 OCR 均支持可配置线程数）
- WPS/Word COM 接口调用通过 `pythoncom.CoInitialize()` 初始化
- COM 操作使用 `threading.Lock` 保证线程安全，`threading.local()` 存储线程状态
- 异常捕获，程序不崩溃（每个模块独立容错）
- 守护线程 + 轮询停止信号，实现立即停止
- SQL 注入防护：LIKE 特殊字符转义（`%`、`_`、`\`）
- 数据库密码环境变量化，不硬编码
- OCR 置信度过滤（阈值 0.7），减少低质量识别的误报
- 图片 MD5 缓存，避免重复 OCR 识别
- 网页增量缓存涉密标记，跳过页面保留链接保证 BFS 完整性

## 十、安全设计
| 安全措施 | 说明 |
|---------|------|
| SQL 注入防护 | `_escape_like()` 转义 LIKE 特殊字符 |
| 密码环境变量化 | `os.environ.get()` 优先读环境变量 |
| Magic Number 校验 | 防止文件扩展名欺骗 |
| 隐藏文件检测 | Windows 隐藏属性 + Linux 点前缀 |
| 压缩包炸弹防护 | 限制嵌套深度 3 层 + 循环引用检测 |
| 加密文件识别 | ZIP 标志位 / OLE2 加密流 / PDF 属性 |
| 输入校验 | URL 格式、数据库名格式、目录存在性 |

## 十一、性能优化
| 优化手段 | 位置 | 效果 |
|---------|------|------|
| `requests.Session()` 连接池 | 网页检查 | 减少 TCP 握手开销 |
| 多线程并行爬取 | 网页检查 | 6 线程并行，速度提升约 6 倍 |
| HTTP 条件请求（增量检测） | 网页检查 | 未变化页面不下载 |
| LIKE 粗筛 + 正则精筛 | 数据库检查 | 减少数据库返回数据量 |
| 跳过数值字段 | 数据库检查 | 减少无用查询 |
| Magic Number 校验 | 文件检查 | 快速判断文件类型 |
| 压缩包并行解压 | 文件检查 | 多个压缩包同时处理 |
| COM 线程锁 | 文件检查 | 保证 COM 安全的同时允许其他任务并行 |
| GPU 加速 OCR | 图片检查 | DirectML/CUDA 加速图片识别 |
| 图片多线程 OCR | 图片检查 | 多张图片同时识别 |
| 图片 MD5 缓存 | 图片检查 | 相同图片跳过 OCR，直接复用结果 |
| OCR 置信度过滤 | 图片检查 | 丢弃低质量识别结果，减少误报 |
